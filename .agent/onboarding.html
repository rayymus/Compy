1. How To Work On This Project

The Iterative Loop (from WORKFLOW.md)

Never stop to handover mid-build. Run this cycle until done:

1Implement — make the change. Keep it minimal: prefer simple over clever.
2Review — spawn a code-reviewer agent on your changes.
3Test — run Python tests + Swift build. Fix breakages immediately.
4Edit workflow files — update AGENTS.md, DEVLOG.md, UPDATES.md, BRAIN.html, clarifications.md as needed. These are your WORKFLOW.md obligations.
5Commit — descriptive message with session number.
6Repeat — go back to step 1 for the next feature.

How To Communicate With The User

  • Ask questions via ask_user tool — multiple choice or free-form. Use when you need decisions.
  • Write status updates in clarifications.md — dated blocks with clear "User:" / "Agent:" labels.
  • Check clarifications.md after every build — the user may have left new instructions.
  • Use suggest_followups at the end of every turn — 3 clickable next steps.
  • Be concise. The user doesn't want walls of text. Cut articles. Minimum for understanding.

How To Build Features

  • Read the spec first. Find it in claude-response*.md or clarifications.md.
  • Gather context. Spawn file-picker and code-searcher agents to find relevant files. Read them.
  • Think before coding. For non-trivial problems, spawn a thinker agent with the relevant file paths.
  • Plan with write_todos. Break work into 4-8 concrete steps with clear completion criteria.
  • Make minimal edits. Prefer str_replace over rewriting whole files. One change at a time.
  • Always test. Run python3 -m pytest compy/daemon/tests -q and cd compy/swift && swift build --quiet after every change batch.
  • Always review. Spawn a code-reviewer agent on your changes.

3. Critical Mistakes — Never Repeat These

1. Never pass None as workspace_root to ripgrep. subprocess.run(cwd=None) → TypeError.

2. to_json(indent=2) breaks NDJSON. Multi-line JSON → lines.last = "}" → every query fails silently.

3. Enrichment + annotation = duplicate Graphify work. Propagate GrepHit.context → RankedHit.structural_context in ALL reasoners.

4. Two Antigravity windows = shared-file race on /tmp/compy-selection.json. Always AX cross-check on hotkey.

5. explain intent requires a selection. Without one, downgrade to fuzzy. Explain patterns MUST come before rationale in _INTENT_RULES.

6. Extension MUST be installed. Without it, AX fallback picks up Terminal → CWD = Desktop/ → wrong workspace.

7. SwiftUI .onDisappear can't animate. Stop timers cleanly; let parent drive transitions.

8. AppleScript fails silently on Electron apps. Use AXUIElement (kAXDocumentAttribute) instead.

9. stage_rename() must resolve the definition node separately from callers — duplicating the callers query means the definition file is never renamed.

10. SwiftUI views inside contentArea are unreachable during .empty phase — shouldShowContent gates the entire subtree. Put overlays (face intro, hint text) in the ZStack layer.

⚠️ Known Limitations

  • Python-only graph — tree-sitter grammar is Python. JS/TS/Rust/Go codebases get grep-only (no explain, no graph_path, no callers).
  • Regex parser — 14 hand-tuned regex rules. Can't truly understand semantics. "What does this do" without "function" → fuzzy.
  • No embedding search — ranking is token overlap (Jaccard + n-gram + rarity). No semantic similarity.
  • Ollama 1.5B only ranks — never generates explanations or summaries. It scores candidates, nothing more.
  • NDJSON fragility — lines.last breaks if anything writes to stdout (debug print, deprecation warning).

5. Tips For Working Efficiently

  • Parallelize. Spawn multiple agents at once — file-pickers, code-searchers, bashers. Don't wait for one before starting the next.
  • Use basher for batch edits. When str_replace is too fiddly (whitespace, large files), use Python scripts via basher.
  • Read, don't guess. Always verify code claims against the actual files. Claude's responses sometimes reference code that doesn't exist yet.
  • Prefer existing patterns. Reuse helper functions, Protocol methods, and model fields. Don't invent new abstractions when existing ones work.
  • Keep changes small. One feature per commit. Easier to review, easier to revert.
  • Don't over-engineer. Compy is a developer tool, not a framework. Simple code that works > elegant code that's fragile.
